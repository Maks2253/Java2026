# 1.1 Контрольні питання (JFC, AWT, Swing)

## 1. Призначення JFC фреймворку і його архітектура

JFC (Java Foundation Classes) — це набір бібліотек для створення графічного інтерфейсу користувача (GUI) у Java. Його основне призначення — забезпечити розробку кросплатформних графічних додатків.

Архітектура JFC включає:

* AWT (Abstract Window Toolkit) — базові графічні компоненти
* Swing — розширена бібліотека GUI
* Java 2D — робота з графікою
* Accessibility API — підтримка доступності
* Drag and Drop — механізм перетягування об’єктів

JFC побудований за принципом подійно-орієнтованої моделі та ієрархії компонентів.

## 2. Переваги бібліотеки Swing у порівнянні з AWT

Основні переваги Swing:

* Більш багатий набір компонентів (JButton, JTable, JTree тощо)
* Платформонезалежний вигляд (Lightweight компоненти)
* Гнучка система налаштування зовнішнього вигляду (Look and Feel)
* Краща архітектура MVC (Model-View-Controller)
* Підтримка подвійної буферизації (менше мерехтіння)

AWT використовує нативні компоненти ОС, а Swing — власні, що робить інтерфейс більш універсальним.

## 3. Приклад структури GUI програми на Java і призначення елементів

Типова структура GUI програми:

* Frame (JFrame) — головне вікно програми
* Panel (JPanel) — контейнер для розміщення компонентів
* Components (JButton, JLabel, JTextField) — елементи інтерфейсу
* Layout Manager — керує розташуванням компонентів
* Event Handlers — обробка дій користувача

Призначення: Frame створює вікно, Panel групує елементи, Components забезпечують взаємодію з користувачем.

## 4. Призначення Containers в AWT/Swing. Що може бути Containers?

Container — це об’єкт, який може містити інші компоненти GUI.

Призначення:

* Організація структури інтерфейсу
* Групування компонентів
* Керування їх розташуванням

Приклади Containers:

* Frame / JFrame
* Panel / JPanel
* Dialog / JDialog
* Applet / JApplet
* ScrollPane / JScrollPane

## 5. Основні методи класу Frame для розроблення GUI

Основні методи:

* setSize(width, height) — встановлення розміру вікна
* setTitle(String) — встановлення заголовка
* setVisible(true) — показ вікна
* add(Component) — додавання компонента
* setLayout(LayoutManager) — встановлення менеджера компонування
* setLocation(x, y) — позиція вікна
* dispose() — закриття вікна

## 6. Призначення об’єкта Panel. Як додати Components до Panel?

Panel (або JPanel) — це контейнер для розміщення та групування компонентів інтерфейсу.

Призначення:

* Логічне групування елементів
* Організація складного інтерфейсу
* Використання різних Layout Manager

Додавання компонентів до Panel:

```java
JPanel panel = new JPanel();
panel.add(new JButton("OK"));
panel.add(new JLabel("Text"));
```

Метод add() використовується для додавання компонентів у панель.

## 7. Призначення об’єктів event в пакеті java.awt.event. Механізм оброблення подій

Об’єкти event — це події, які виникають під час взаємодії користувача з інтерфейсом (клік миші, натискання клавіші тощо).

Механізм оброблення подій:

1. Користувач виконує дію
2. Генерується об’єкт події (Event)
3. Подія передається слухачу (Listener)
4. Listener обробляє подію через відповідний метод

Це називається подійно-орієнтованою моделлю програмування.

## 8. Призначення інтерфейсів типу XxxListener. Типи і приклади

XxxListener — це інтерфейси для обробки конкретних подій.

Основні типи:

* ActionListener — натискання кнопки
* MouseListener — події миші
* KeyListener — події клавіатури
* WindowListener — події вікна
* FocusListener — фокус елементів

Приклад використання:

```java
JButton button = new JButton("Click");
button.addActionListener(new ActionListener() {
    public void actionPerformed(ActionEvent e) {
        System.out.println("Кнопку натиснуто");
    }
});
```

Метод addActionListener() підключає обробник подій до компонента.
